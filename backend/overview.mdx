## What is the Backend?

The `auditor-backend` is a **FastAPI** application that serves as the single API for both frontend apps. It handles authentication, website management, audit orchestration, billing, RUM, reports, and all business logic.

<Columns cols={2}>
  <Card title="Framework" icon="bolt">FastAPI on Python 3.12 · Uvicorn ASGI server</Card>
  <Card title="Database" icon="database">PostgreSQL via SQLAlchemy 2 (async) · asyncpg driver · Alembic migrations</Card>
  <Card title="Cache" icon="memory">Redis via aioredis — sessions, permission cache, scan status</Card>
  <Card title="Auth" icon="lock">JWT RS256 · Google OAuth · Microsoft OAuth · AWS Cognito</Card>
  <Card title="Payments" icon="credit-card">Stripe — subscriptions, checkout, webhooks</Card>
  <Card title="AI" icon="sparkles">Google Gemini 2.0 Flash — accessibility fix generation</Card>
</Columns>

| URL | Purpose |
|-----|---------|
| `http://localhost:8000` | API base (dev) |
| `http://localhost:8000/api/openapi.json` | Swagger / OpenAPI spec |

## Application Startup Lifecycle

<Steps>
  <Step title="Entrypoint">
    `python -m auditor_app` calls `uvicorn.run()` with `factory=True`, pointing at `auditor_app.web.application:get_app`.
  </Step>
  <Step title="App factory runs">
    `get_app()` creates the FastAPI instance, registers `CORSMiddleware` and `SessionMiddleware`, then mounts the master router at `/api`.
  </Step>
  <Step title="Database pool created">
    An async SQLAlchemy engine is created with `pool_size=20`, `max_overflow=10`. A session factory is stored on `app.state`.
  </Step>
  <Step title="OpenTelemetry initialised">
    If `AUDITOR_APP_OPENTELEMETRY_ENDPOINT` is set, FastAPI, SQLAlchemy, and Redis instrumentation is activated and connected to the OTLP exporter.
  </Step>
  <Step title="Redis pool created">
    An async Redis connection pool is created and stored on `app.state.redis_pool`.
  </Step>
  <Step title="Redis cache warmed">
    Four datasets are loaded from PostgreSQL and cached in Redis on every restart: role→permission mappings, accessibility audit rules, scan regions, and uptime monitor API tokens.
  </Step>
</Steps>

**On shutdown:** database pool disposed → Redis connections closed → OpenTelemetry flushed.

## Configuration

All settings live in `auditor_app/settings.py` using **pydantic-settings**. Every field maps to an environment variable prefixed with `AUDITOR_APP_`.

Copy `.env.sample` → `.env` and fill in your values. Key variables for local dev:

```ini
AUDITOR_APP_HOST=127.0.0.1
AUDITOR_APP_PORT=8000
AUDITOR_APP_RELOAD=True
AUDITOR_APP_ENVIRONMENT=dev

AUDITOR_APP_DB_HOST=localhost
AUDITOR_APP_DB_PORT=5432
AUDITOR_APP_DB_USER=admin
AUDITOR_APP_DB_PASS=admin
AUDITOR_APP_DB_BASE=auditor_dev

AUDITOR_APP_REDIS_HOST=localhost
AUDITOR_APP_REDIS_PORT=6379

AUDITOR_APP_PRIVATE_KEY_FILE_PATH=cert/private.key
AUDITOR_APP_PUBLIC_KEY_FILE_PATH=cert/public.key
AUDITOR_APP_ALGORITHM=RS256
```

See the [dev setup guide](/onboarding/dev-setup) for the full variable reference.

## Middleware Stack

| Middleware | Purpose |
|-----------|---------|
| `CORSMiddleware` | Allows all origins, credentials, methods, and headers |
| `SessionMiddleware` | Server-side session support (required for OAuth flows) |
| SlowAPI | Per-IP rate limiting on auth and registration endpoints |

## Dependency Injection Pattern

FastAPI's `Depends()` injects these into every route handler that needs them:

```python
@router.get("/")
async def get_websites(
    db: AsyncSession = Depends(get_db_session),      # DB session per request
    redis_pool = Depends(get_redis_pool),            # Redis connection pool
    current_user: dict = Depends(verify_token),      # Decoded JWT payload
):
    ...
```

Two auth dependencies are available:

| Dependency | Requires |
|-----------|---------|
| `verify_token` | Valid, non-expired JWT in Redis |
| `verify_token_needs_email_verification` | Valid JWT **and** `email_verified = True` |

## DAO Pattern

All database queries go through **DAO classes** in `db/dao/`. Route handlers must never contain raw SQLAlchemy queries.

```python
from auditor_app.db.dao.website_dao import WebsiteDAO

websites = await WebsiteDAO().get_by_org_id(
    db, org_id=current_user["organisation_id"]
)
```

## Running Alembic Migrations

```bash
# Apply all pending migrations
poetry run alembic upgrade head

# Create a migration after changing a model
poetry run alembic revision --autogenerate -m "add_website_rum_status"

# Roll back one migration
poetry run alembic downgrade -1

# Check current state
poetry run alembic current
```

Migration files live in `auditor_app/db/migrations/versions/`.

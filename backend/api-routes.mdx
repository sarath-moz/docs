title: "API Routes"

---

All routes are mounted under `/api`. The base URL in production is `https://api.webyes.com`.

**Authentication:** Most endpoints require `Authorization: Bearer <token>` header.
**Rate limiting:** Applied to sensitive endpoints (10 requests/minute by IP).

## Auth — `/api/auth`

### Standard Auth
| Method | Path | Body | Auth | Description |
|--------|------|------|------|-------------|
| `POST` | `/api/auth/login` | `{email, password}` | None, rate-limited | Login with email/password — returns JWT |
| `POST` | `/api/auth/logout` | — | `verify_token` | Invalidate current JWT in Redis |
| `POST` | `/api/auth/forgot-password` | `{email}` | None | Send password reset email |
| `POST` | `/api/auth/reset-password` | `{token, new_password}` | None | Complete password reset |
| `POST` | `/api/auth/refresh-token` | — | Bearer refresh_token | Get new access token |
| `POST` | `/api/auth/validate-token` | — | `verify_token` | Check token validity |

### Google OAuth

## Response Format

All endpoints return consistent JSON:

```json
{
  "success": true,
  "data": { ... }
}
```

On error:
```json
{
  "success": false,
  "message": "Error description",
  "detail": "..."
}
```

HTTP status codes follow REST conventions: `200` (OK), `201` (Created), `400` (Bad Request), `401` (Unauthorized), `403` (Forbidden), `404` (Not Found), `422` (Validation Error), `429` (Rate Limited).

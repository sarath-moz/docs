---
title: "Email Templates"
description: "Complete catalog of all 35+ transactional email templates."
---

All emails are React components built with `@react-email/components`, rendered to HTML server-side, and sent via **Resend**. Pass the template name as the `template` field in `POST /api/v1/email`.

**Brand colours used across all templates:**

| Token | Hex |
|---|---|
| Primary blue | `#0B66E4` |
| Body text | `#2E3240` |
| Heading text | `#242424` |
| Background | `#eef1fb` |
| Footer background | `#f7f7f8` |
| Top border accent | `6px solid #0B66E4` |

## User Onboarding & Auth

<AccordionGroup>
  <Accordion title="verify — Email OTP verification">
    Sends a 6-digit OTP for email address verification.

    **Template name:** `verify`

    **Required `data` fields:**

    | Field | Type | Description |
    |---|---|---|
    | `otp` | string | The 6-digit OTP code |
    | `name` | string | User's first name |
    | `appUrl` | string | Link back to the app |

    **Subject:** `"Verify your email address"`
  </Accordion>

  <Accordion title="verify-email-link — Link-based email verification">
    Sends a clickable verification link instead of an OTP code.

    **Template name:** `verify-email-link`

    **Required `data` fields:**

    | Field | Type | Description |
    |---|---|---|
    | `verificationLink` | string | Full verification URL |
    | `name` | string | User's first name |
  </Accordion>

  <Accordion title="reset — Password reset">
    Sends a password reset link.

    **Template name:** `reset`

    **Required `data` fields:**

    | Field | Type | Description |
    |---|---|---|
    | `resetLink` | string | Full password reset URL |
    | `name` | string | User's first name |

    **Subject:** `"Reset your password"`
  </Accordion>

  <Accordion title="invite — Team invitation">
    Invites a new user to join an organisation.

    **Template name:** `invite`

    **Required `data` fields:**

    | Field | Type | Description |
    |---|---|---|
    | `inviteLink` | string | Invitation acceptance URL |
    | `name` | string | Inviter's name |
    | `organisationName` | string | Organisation name |
  </Accordion>

  <Accordion title="transfer-site — Site transfer notification">
    Notifies a user that a website has been transferred to their account.

    **Template name:** `transfer-site`

    **Required `data` fields:**

    | Field | Type | Description |
    |---|---|---|
    | `websiteUrl` | string | The transferred website |
    | `fromEmail` | string | Original owner's email |
  </Accordion>
</AccordionGroup>

## Trial Lifecycle

These emails are sent automatically during the trial period progression.

| Template name | When sent |
|---|---|
| `welcome-trial-suite` | Immediately on suite trial start |
| `welcome-trial-accessibility` | Immediately on accessibility trial start |
| `trial-reminder` | During trial period as a general reminder |
| `trial-expires-soon` | 3 days before trial expires |
| `three-day-trial-end` | Final 3-day warning |
| `trial-expires-today` | On the day of expiry |
| `trial-expired` | After trial has ended |
| `credit-expired-trial` | When trial scan credits are exhausted |

**Required `data` fields (all trial emails):**

| Field | Type | Description |
|---|---|---|
| `name` | string | User's first name |
| `appUrl` | string | Link to the app |
| `daysRemaining` | number | Days left (where applicable) |

## Uptime Monitoring

<AccordionGroup>
  <Accordion title="site-down — Website down alert">
    Sent immediately when an uptime monitor detects a website is unreachable.

    **Template name:** `site-down`

    | Field | Type | Description |
    |---|---|---|
    | `websiteUrl` | string | Affected website URL |
    | `detectedAt` | string | ISO 8601 timestamp of detection |
    | `name` | string | User's name |
  </Accordion>

  <Accordion title="site-up — Website recovered">
    Sent when the uptime monitor confirms a previously down website is accessible again.

    **Template name:** `site-up`

    | Field | Type | Description |
    |---|---|---|
    | `websiteUrl` | string | Recovered website URL |
    | `downtimeDuration` | string | How long the site was down (e.g. `"14 minutes"`) |
    | `name` | string | User's name |
  </Accordion>
</AccordionGroup>

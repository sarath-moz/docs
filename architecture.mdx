## Platform Overview

The Webyes Auditor platform has three independently deployable layers: two React frontend apps sharing a common library, and one FastAPI backend serving them both.

<Columns cols={3}>
  <Card title="full-suite" icon="browser" href="/frontend/full-suite">
    React + Vite SPA · Port 5173 · Full feature set including billing and team management
  </Card>
  <Card title="a11y" icon="universal-access" href="/frontend/a11y">
    React + Vite SPA · Port 5174 · Accessibility-only standalone product
  </Card>
  <Card title="auditor-backend" icon="server" href="/backend/overview">
    FastAPI · Port 8000 · Single API serving both apps
  </Card>
</Columns>

Both apps share code through `packages/common` (UI library) and `packages/focus-order` (accessibility tool), and both talk to the same backend over HTTPS REST.

---

## Component Responsibilities

| Component | Responsibility |
|-----------|---------------|
| **full-suite** | Accessibility, performance, SEO, uptime, RUM, billing, and team management |
| **a11y** | Accessibility-only auditing with simplified onboarding |
| **packages/common** | Shared React components, types, utilities, hooks, and 150+ icons |
| **packages/focus-order** | Keyboard focus-order visualiser (iframe + Shadow DOM support) |
| **auditor-backend** | Auth, user management, scans, issues, billing, RUM, reports |
| **PostgreSQL** | Persistent data store for all entities |
| **Redis** | JWT session store, permission cache, real-time scan status |

## Key Data Flows

<AccordionGroup>
  <Accordion title="1. User Authentication">
    <Steps>
      <Step title="Login request">
        Browser sends `POST /api/auth/login` with email and password.
      </Step>
      <Step title="Token issued">
        Backend validates credentials, signs a JWT (RS256), stores it in Redis, and returns `{ access_token }`.
      </Step>
      <Step title="Authenticated requests">
        Frontend stores the token in a cookie. All subsequent API calls include `Authorization: Bearer <token>`.
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="2. Website Audit Scan">
    <Steps>
      <Step title="Trigger scan">
        User initiates a scan → `POST /api/website-audit/`. Backend creates a `WebsiteAudit` record with `status=in_progress`.
      </Step>
      <Step title="Scanner called">
        Backend calls the external Lighthouse Scanner API with the website URL and a webhook callback URL.
      </Step>
      <Step title="Frontend polls">
        Frontend polls `GET /api/website-audit/latest-audit-status/` every few seconds to track progress.
      </Step>
      <Step title="Results received">
        Scanner finishes and POSTs results back to the backend webhook. Backend saves `LighthouseAudit` records and stores full JSON in S3.
      </Step>
      <Step title="Results available">
        `WebsiteAudit.status` → `completed`. Frontend fetches results from `/api/results/`.
      </Step>
    </Steps>
  </Accordion>

  <Accordion title="3. Report Generation">
    <Steps>
      <Step title="Request report">User requests a PDF report → `GET /api/report/`.</Step>
      <Step title="Data compiled">Backend aggregates audit data across all pages and metrics.</Step>
      <Step title="S3 URL returned">A presigned S3 URL is returned — valid for 2 minutes. Frontend opens it for download.</Step>
    </Steps>
  </Accordion>

  <Accordion title="4. Billing (Stripe)">
    <Steps>
      <Step title="Select plan">User picks a subscription plan → `POST /api/stripe/create-checkout-session/`.</Step>
      <Step title="Stripe checkout">Backend creates a Stripe Checkout Session and returns the redirect URL. Frontend redirects the user.</Step>
      <Step title="Payment completed">Stripe sends a webhook to `POST /api/stripe/webhook`. Backend verifies the signature, creates `UserPlanPurchase`, and updates subscription limits.</Step>
      <Step title="Redirect">User is sent to the success or failure URL configured in env vars.</Step>
    </Steps>
  </Accordion>

  <Accordion title="5. AI Fix Suggestion (Gemini)">
    <Steps>
      <Step title="Request fix">Frontend sends `POST /api/solutions/` with the issue rule ID and HTML snippet.</Step>
      <Step title="Prompt built">Backend selects the right prompt template for the issue type and calls Gemini 2.0 Flash.</Step>
      <Step title="Fix stored">AI response `{ current_code, correct_code, explanation }` is cached in S3 for 180 days.</Step>
      <Step title="Fix returned">A presigned S3 URL is returned to the frontend.</Step>
    </Steps>
  </Accordion>
</AccordionGroup>

## Tech Stack at a Glance

<Columns cols={2}>
  <Card title="Frontend" icon="browser">
    | Layer | Tech |
    |-------|------|

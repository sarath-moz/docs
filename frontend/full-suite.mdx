## Purpose

The **full-suite** app is the primary Webyes Auditor dashboard. It gives users:
- Accessibility, performance, SEO, and quality audit results
- Scan scheduling and history
- Real User Monitoring (RUM) data
- Uptime monitoring
- Team and organization management
- Subscription billing (Stripe)
- PDF report generation
- Google Search Console integration

**Dev URL:** http://localhost:5173
**Staging:** https://app.staging.webyes.com
**Production:** https://app.webyes.com

---

## Entry Point

```
apps/full-suite/src/
├── main.tsx          ← Mounts React, wraps with BugsnagProvider
├── App.tsx           ← Creates TanStack Router, passes context
└── routeTree.gen.ts  ← AUTO-GENERATED — never edit manually
```

`App.tsx` creates the router and injects context (user email) available to all routes:

```typescript
const router = createRouter({
  routeTree,
  context: { userEmail, setEmail },
})
```

## Routing Structure

Routes live in `src/routes/`. The folder structure maps directly to URL paths.

### Route Conventions

| Pattern | Meaning |
|---------|---------|
| `__root.tsx` | Root layout wrapping every page |
| `(auth)/` | Route group — no URL segment, groups auth pages |
| `(protected)/` | Route group — no URL segment, groups gated pages |
| `_layout.tsx` | Nested layout component |
| `.lazy.tsx` suffix | Code-split — loaded only when visited |
| `$metrics` in filename | Dynamic URL parameter |

## State Management

### Client State (Zustand)
Location: `src/store/client/`

| Store | File | What it holds |
|-------|------|---------------|
| `useScan` | `scan.ts` | Active scan schedule state |
| `useDialogue` | `dialogue.ts` | Dialog/modal open states |
| `useAuditResult` | `auditResult.ts` | Cached audit result data |
| `useSolutions` | `solutions.ts` | AI-generated solution data |
| `useTidio` | `tidioHandle.ts` | Tidio chat widget state |
| `useBilling` | `billing.ts` | Billing flow state |
| `useUtilKeys` | `utilKeys.ts` | Utility key state |
| `useRum` | `rum.ts` | RUM config and filters |

**Store Pattern:**
```typescript
import { create } from "zustand"

interface ScanState {
  scheduleScan: ScanScheduleResponse | null
}
interface ScanActions {
  setScheduleScan: (v: ScanScheduleResponse | null) => void
}

const useScan = create<ScanState & ScanActions>((set) => ({
  scheduleScan: null,
  setScheduleScan: (scheduleScan) => set({ scheduleScan }),
}))

export default useScan
```

## API Layer

**Config:** `src/lib/api.ts`
```typescript
const API_URL = import.meta.env.VITE_API_URL
// All requests → Bearer token from "access_token" cookie
```

| File | Responsibility |

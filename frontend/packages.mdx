## packages/common

The `common` package is a **shared React component library** consumed by both the `full-suite` and `a11y` apps. It lives in `packages/common/` and is referenced via pnpm workspace protocol (`"common": "workspace:*"`).

<Warning>
  Never copy-paste components between apps. If a component is needed in both, it belongs in `packages/common`. If it's specific to one app, keep it in that app's `src/components/`.
</Warning>

### Styling in common

The common package doesn't have its own Tailwind config. It relies on the **consuming app's Tailwind** setup. The apps include common's source in their Tailwind content paths:

```javascript
// apps/full-suite/tailwind.config.js
content: [
  "./src/**/*.{js,ts,jsx,tsx}",
  "../../packages/common/src/components/**/*.{js,ts,jsx,tsx}",
]
```

All Tailwind classes used in `common` components must use the `wy-` prefix (set in each app's Tailwind config).

## packages/focus-order (`@webyes/focus-order`)

The `focus-order` package is a **standalone accessibility library** that visualises keyboard focus order in web pages. It's published as a dual ESM/CJS npm package.

**Package name:** `@webyes/focus-order`
**Reference:** `"@webyes/focus-order": "workspace:*"` in app dependencies

---

### What It Does

When called, it:
1. **Scans the DOM** for all focusable elements (links, buttons, inputs, etc.)
2. **Supports iframes** — crawls into iframe documents
3. **Supports Shadow DOM** — finds elements inside shadow roots
4. **Detects screen-reader-only elements** — flags elements not visually visible
5. **Renders numbered badges** next to each focusable element
6. **Draws SVG lines** connecting badges in tab order
7. **Repositions badges** on scroll and resize

---

### Main Export

```typescript
import { focusableElementRender } from "@webyes/focus-order"

const { handleRepaint, docsToWatch, focusTargets } = focusableElementRender({
  doc: document,          // The document to scan
  excludeClasses: [],     // Optional: CSS classes to ignore
})

// docsToWatch: all Documents to observe (main + iframes)
// focusTargets: array of FocusableElementMeta
// handleRepaint(): call to re-render badges after DOM changes
```

---

### Types

```typescript
type FocusableElementMeta = {
  element: HTMLElement     // The focusable DOM element
  doc: Document            // The document it lives in
  label: string            // Display label (text content or aria-label)
  selector: string         // CSS selector for the element
  target: string[]         // Connected element selectors
}

type FocusTarget = {
  element: HTMLElement
  doc: Document
}
```

---

### Utility Functions

| Function | Purpose |
|----------|---------|
| `getFocusableElementsMeta(doc, excludeClasses?)` | Get all focusable elements with metadata |
| `findFocusableElements(doc, rootElement?, excludeClasses?)` | Find raw focus targets |
| `createFocusOrderBadgeGlobal(hostDoc, elementDoc, index, element)` | Render a numbered badge |
| `drawFocusOrderLines(doc)` | Draw SVG connecting lines |
| `getAllAccessibleDocuments(rootDoc)` | Get all iframe documents |
| `repositionFocusOrderBadges(doc)` | Update badge positions after scroll/resize |
| `computeCenterRelativeTo(hostDoc, elementDoc, element)` | Calculate element center point |

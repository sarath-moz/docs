## Base URLs

| Environment | URL |
|-------------|-----|
| Local dev | `http://localhost:8000` |
| Staging | `https://api.staging.webyes.com` |
| Production | `https://api.webyes.com` |

Interactive docs (Swagger UI) are available at `/api/openapi.json` on any running instance.

---

## Authentication

All protected endpoints require a **JWT bearer token** obtained from `POST /api/auth/login`.

```http
Authorization: Bearer eyJhbGciOiJSUzI1NiIs...
```

Tokens are RS256-signed JWTs stored in Redis. They expire after **7 days**. Logging out immediately invalidates the token in Redis regardless of the expiry time.

<Note>
  The Stripe webhook endpoint (`POST /api/stripe/webhook`) is the only endpoint that uses **Stripe signature verification** instead of JWT. Do not send a bearer token to that endpoint.
</Note>

## Response Format

Every endpoint returns the same JSON envelope:

```json
{ "success": true,  "data": { ... } }
```

```json
{ "success": false, "message": "Error description" }
```

| Status Code | Meaning |
|-------------|---------|
| `200` | OK |
| `201` | Created |
| `400` | Bad request / validation error |
| `401` | Missing or invalid token |
| `403` | Valid token but insufficient permission |
| `404` | Resource not found |
| `422` | Unprocessable entity (Pydantic validation) |
| `429` | Rate limit exceeded |

---

## Rate Limiting

Sensitive endpoints are rate-limited to **10 requests per minute per IP** via SlowAPI:

- `POST /api/auth/login`
- `POST /api/user/` (registration)
- `POST /api/user/verify-email/`
- `POST /api/user/verify-email-otp/`
- `POST /api/user/verify-email-link/`
- `POST /api/user/invite-user/`

---

## Permissions

Some endpoints require specific **permission slugs** on the authenticated user's role in addition to a valid JWT.

| Slug | Required for |
|------|-------------|
| `settings:website:add` | Add a website |
| `settings:website:delete` | Delete a website |
| `settings:website:transfer` | Transfer a website |
| `settings:team:add` | Invite team members |
| `scan:full_scan:add` | Start an audit scan |
| `settings:organisation:edit` | Edit organisation details |

If the permission is missing the endpoint returns `HTTP 403`.

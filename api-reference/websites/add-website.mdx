---
title: "Add Website"
openapi: "POST /api/website/"
description: "Register a new website to the organisation for auditing."
---
**`POST`**
```
/api/website/
```


Adds a website to the authenticated user's organisation. The website will be available for scans immediately after creation. Requires the `settings:website:add` permission.

## Request Body

<ParamField body="website_url" type="string" required>
  The full URL of the website (e.g. `https://example.com`). Must be a valid URI.
</ParamField>

<ParamField body="website_name" type="string">
  A friendly display name for the website. Defaults to the domain if omitted.
</ParamField>

<ParamField body="region_id" type="string">
  UUID of the scan server region to use. Get available regions from [List Regions](/api-reference/monitoring/list-regions).
</ParamField>

## Response

<ResponseField name="success" type="boolean">
  `true` when the website has been created.
</ResponseField>

<ResponseField name="data" type="object">
  The newly created website object with `id`, `website_url`, `website_name`, `status`, `organisation_id`.
</ResponseField>

<RequestExample>
```json
{
  "website_url": "https://example.com",
  "website_name": "Example Site",
  "region_id": "b1c2d3e4-f5a6-7890-bcde-f12345678901"
}
```
</RequestExample>
